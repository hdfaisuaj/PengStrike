# PengStrike 安全策略

> ⚠️ **重要安全警告**
>
> 本项目**仅设计用于本地开发和学习环境**（如 `127.0.0.1` 或受信任的内网）。
>
> **请勿将本项目部署在公网或生产环境！**
>
> 主要安全风险：
> 1. **WebSocket 明文传输**：默认使用 `ws://`（非 TLS 加密），所有数据（包括 sudo 密码、命令输出、会话数据）以明文形式传输，可被网络嗅探工具（如 Wireshark）轻易捕获
> 2. **sudo 密码处理**：后端执行需要 sudo 权限的命令时，会通过 WebSocket 请求密码。在 `ws://` 下，密码以明文传输
> 3. **缺少 WebSocket 身份认证**：当前版本 WebSocket 连接缺乏身份验证机制，任何能访问后端地址的人都可以连接
> 4. **CORS 配置宽松**：开发模式下 CORS 允许所有来源（`*`），生产环境必须严格配置
> 5. **会话管理简单**：会话 ID 生成简单，缺乏适当的会话过期和撤销机制
>
> **如果必须在非本地环境运行：**
> - 必须启用 TLS（使用 `wss://` 和 `https://`）
> - 必须配置反向代理（如 Nginx）并启用 TLS
> - 必须添加 WebSocket 身份认证（如 JWT token）
> - 必须正确配置 CORS 策略（白名单模式）
> - 建议修改 sudo 密码传输机制（如改用临时令牌或加密通道）
>
> **推荐部署方式：**
> - 本地运行：前端 `http://localhost:5173`，后端 `http://127.0.0.1:8000`
> - 内网运行：确保网络隔离，使用防火墙限制访问

## 安全声明

PengStrike 平台重视安全性，所有功能均在合法授权的环境中使用。未经授权使用本平台进行渗透测试可能违反法律，使用者需自行承担所有法律责任。

## Web 安全特性

### WebSocket 安全
- **实时数据传输**：通过 WebSocket 实现实时日志和命令执行结果推送
- **连接管理**：支持自动重连和心跳检测，确保连接稳定性
- **数据加密**：在 HTTPS 环境下，确保数据传输加密

### 命令执行安全
- **工具白名单**：仅允许预定义的 Kali 工具执行
- **输入验证**：对命令输入进行严格验证，防止注入攻击
- **路径限制**：禁止通过全路径执行命令，避免绕过安全机制

### 会话安全
- **会话隔离**：每个用户会话独立，确保数据不被泄露
- **状态记录**：所有会话状态变更均进行记录和审计

## 安全建议

### 开发者注意事项
- **CORS 配置**：在生产环境中严格限制 CORS 来源
- **HTTPS 要求**：确保所有 WebSocket 和 REST API 使用 HTTPS 传输
- **定期更新**：及时更新依赖库，修复已知漏洞

### 使用者注意事项
- **合法使用**：仅在合法授权的环境中使用本平台
- **数据保护**：确保所有测试数据和结果保持机密
- **日志审计**：定期审查日志，确保没有未经授权的访问

## 报告安全漏洞

如果发现安全漏洞，请按照以下步骤报告：

1. **不公开披露**：请勿公开披露任何安全漏洞信息
2. **私有渠道报告**：通过邮件或 GitHub Security Advisory 提交漏洞报告
3. **详细信息**：提供漏洞的详细描述和复现步骤

我们承诺在 72 小时内回复，并在修复后公开致谢。

## 已知安全限制

- **WebSocket 认证**：当前版本缺乏 WebSocket 连接身份验证，生产环境应添加认证机制
- **CORS 限制**：开发模式下允许所有来源，生产环境应严格配置
- **命令注入**：依赖字符串匹配，可能存在绕过风险，正在持续更新检测逻辑

## 更新周期

- **安全更新**：发现漏洞后 7 天内发布修复
- **常规更新**：每季度发布
- **紧急更新**：高危漏洞 24 小时内发布热修复
